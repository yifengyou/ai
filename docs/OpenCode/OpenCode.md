# OpenCode (智能体)

* 官网: <https://opencode.ai/>
* 源码：<https://github.com/anomalyco/opencode>

OpenCode 是一款开源、免费且模型无关的 AI 编程智能体（Coding Agent）。它旨在让开发者在不离开终端（Terminal）环境的前提下，高效地完成代码生成、理解、调试与重构等任务。

你可以把它想象成一个坐在你终端里的 AI 同事，它能够理解整个项目，并根据你的自然语言指令来编写代码、修复 Bug 或进行重构。

### 🤔 核心定位

OpenCode 本身并非一个 AI 模型，而是一个连接你与各种大语言模型（LLM）的“调度层”或客户端。它基于 MIT 许可证完全开源，强调“终端优先、多模型、隐私安全”的理念。

### ✨ 主要特性

*   **模型无关 (Model-Agnostic)**
    OpenCode 最大的特点是支持自由切换超过 75 种 AI 模型。你可以使用 OpenAI 的 GPT、Anthropic 的 Claude、Google 的 Gemini，也可以通过 Ollama 等工具连接本地部署的开源模型（如 Llama 3、Qwen）。这让你摆脱了对单一厂商的依赖，可以根据任务需求和成本灵活选择模型。

*   **终端原生 (Terminal-First)**
    它专为命令行环境设计，提供了一个美观流畅的终端用户界面（TUI）。对于习惯使用终端的开发者来说，这意味着无需在编辑器和浏览器之间来回切换，可以保持专注，大幅提升工作效率。

*   **强大的智能体 (Agent) 能力**
    OpenCode 具备自主规划和执行的能力，不仅能生成代码片段，还能直接读写文件、执行命令、管理项目。它内置了多种工具，如 BashTool（执行命令）、EditTool（编辑文件）、GrepTool（搜索内容）等，使其成为一个真正的“行动者”。

*   **隐私优先 (Privacy-First)**
    采用“隐私优先”的架构设计，默认不会存储或上传你的任何代码和上下文数据。所有数据传输仅发生在你与所选的 LLM 服务商之间，确保了代码的私密性。

*   **多形态支持**
    除了核心的命令行（CLI）和终端界面（TUI），OpenCode 还提供桌面应用（Beta版）以及 VS Code、Cursor 等主流 IDE 的插件，可以适配不同开发者的使用习惯。

### 🛠️ 核心功能

1.  **Plan / Build 双模式**
    *   **Plan 模式 (规划)**：AI 以“只读”方式分析你的项目，生成开发计划或重构方案，但不会直接修改代码。这非常适合进行架构讨论和方案评审。
    *   **Build 模式 (构建)**：在确认计划后，AI 获得“写入”权限，根据规划自动执行代码修改、文件创建等操作。这种“先规划，后执行”的流程让开发过程更安全可控。

2.  **深度开发工具集成**
    OpenCode 内置了对语言服务器协议（LSP）的支持，能够深度理解代码的语法和结构，提供更精准的代码分析和诊断。同时，它还支持与远程和本地的 MCP 服务器集成，扩展其能力边界。

3.  **多智能体协作 (Multi-Agent)**
    可以通过 `Oh My OpenCode` 等元框架，定义和编排多个专业化的 AI 智能体协同工作。例如，一个智能体负责代码审查，另一个负责性能分析，共同完成复杂的开发任务。
